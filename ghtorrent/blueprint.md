# V2（数据精炼项目 · 基础版）

## 原始需求

### 沟通背景

基于数万级样本，衔接公开数据源，匹配第三方平台信息，最终产出每日活动面板（daily activity panel）。

### 数据来源

| 来源 | 说明 |
|------|------|
| 主数据库 | `{{mysql_dump}}` — 公开数据项目抓取的全量数据 |
| ID 列表 | `{{user_list}}` — 数万级用户 ID |

处理流程：本地解压 → 提取关键 CSV → 用标识列表匹配用户表获取 ID → 以 ID 为核心过滤其他表。

### 输出变量（N 列）

| 变量 | 描述 |
|------|------|
| col_01 | 去标识化的数字标识符 |
| col_02 | 账户标识名 |
| col_03 | 活动观察日期 |
| col_04 | 维度 A 的日计数 |
| col_05 | 维度 B 的日计数 |
| col_06 | 维度 C 的日计数 |
| col_07 | 维度 C 子类型的日计数 |
| col_08 | 维度 D 的日计数 |
| col_09 | 维度 D 活跃子类型的日计数 |
| col_10 | 控制变量（二进制指示器） |

> 注意：实际变量选择与口径定义属于客户计量建模核心资产，此处已脱敏为通用占位符。

### 精炼管道（11 步 → 14 步）

| 阶段 | 步骤 |
|------|------|
| 基础设施 | 环境搭建与数据下载 → 数据解压与提取 |
| 数据筛选 | 用户匹配 → 数据子集筛选（多步骤） |
| 指标计算 | 多维度日聚合计数 |
| 面板合并 | 面板合并 |
| 交付 | 数据验证与交付 |

### 上云方案

{{云服务器规格}}（可配置：vCPU / 内存 / 数据盘 / 区域）。

### 交付物

- CSV 格式面板数据，按主键排序，严格 N 列
- 配套 1 份《数据处理说明》

# V3（基于 V2 迭代）

在 V2 基础上延续做数据精炼，以下仅列出与 V2 的差异。

## 需求变更

### 用户名单变更

数据沿用细分名单，从 **数万级** 缩减至 **数万级** 个用户。

输入文件：`{{user_list_v2}}`

### 新增变量

在 V2 基础上扩充维度 D 和维度 C 的细分口径，新增若干变量。具体拆分策略和判定标准属于客户研究设计，此处已脱敏。

### 输出变量（M 列）

V2 的若干基础变量保留，部分原始变量被细分变量替代，共 M 列。

### 管道变更（14 步 → 16 步）

管道中涉及细分的步骤按新口径拆分，步骤数从 14 步增至 16 步。

### 交付物变更

- CSV 格式面板数据，按主键排序，严格 **M** 列
- 《数据处理说明》同步更新，补充新变量的口径定义
